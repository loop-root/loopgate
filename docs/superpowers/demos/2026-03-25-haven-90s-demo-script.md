# Haven 90-second demo script

**Purpose:** Launch artifact for v1 — show security through UX, not architecture slides.  
**Plan:** `docs/superpowers/plans/2026-03-25-master-implementation-plan.md` Task 11.

## Beat sheet (~90s)

| Seconds | Beat | What to show |
|--------:|------|----------------|
| 0–5 | Install | Drag **Haven** from `.dmg` to **Applications** (or open signed app). |
| 5–20 | First-run | Model choice (Ollama local or cloud) + **grant Downloads** (or Skip). Finish setup. |
| 20–30 | Hello | Morph greets; user asks to **organize Downloads** (or similar). |
| 30–45 | Plan | Morph scans / proposes a **concrete plan** (folders, counts). |
| 45–55 | Approval | **Approval card** — user reads scope, approves. |
| 55–65 | Result | Files move; Morph gives a **short, warm confirmation** + facts. |
| 65–80 | Security | Open **Security** (or Activity): audit trail, permissions, what was allowed. |
| 80–90 | Memory | Quit Haven, reopen — **name / task** still there (continuity). |

## Capture notes

- Resolution: **1920×1080** minimum; clean desktop; notifications off; no personal filenames visible.
- Voiceover optional; **on-screen text callouts** are enough for v1.
- Edit to **~90s**; cut hesitation; keep the security panel legible for one beat.

## Post-recording checklist

- [ ] First-run path matches current wizard (model + folders only; rest in Settings).
- [ ] Approval moment is clearly visible.
- [ ] No raw secrets or API keys on screen.
